Fall Prevention

Organization: Vertically Integrated Projects @ Georgia Tech

Team: Brain Trauma Assessment Protocols

Role: UX Engineer, Project Manager

Design Tools: Figma

Development Tools: ReactJS, TypeScript

Project Description: A mobile application designed to assist healthcare clinicians, older adults, and individuals with cognitive impairments in planning and achieving fall prevention goal.


Identify

We conducted a interview via an email chain with our client to better understand what she was looking for in the administrator portal of the Fall Prevention Application. From the conversation, we were able to gather that she wanted 3 major functions to appear: Login capability, New User Account feature, and the ability to track the performance of their respective patients.

    Admin Login
  • Required Inputs:
    • admin email
    • admin password
  • A page for admin to manage users
  • A profile page for users to view/edit their login credentials
    New User Account Page
  • Required Inputs
    • name
    • email address
    • age
  • User verification during installation
  • Create user button
    Task Performance Page
  • Required Inputs
    • Progress Indication in the form of graphs
    • Patient Messages
    • Patient Notes
  • Notifications system

Ideate

Over the course of one month, I took the user requirements that we were given and used Figma to create the mockup of the web application. Because the iOS application has a color scheme of blues and whites, I decided to stick to that color palette to maintain consistency across the platforms. The images displayed show the primary screens for the web application.


Development

To develop the web-application, I am using the ReactJS framework and TypeScript. Currently the website is under-development and I will be workign on this over the course of the next 2-3 months.

User Testing

We conducted user testing of the web application by sending our client the mockups that were created, and got some insight into her expectations for the end result.

    User feedback:
  • Logo looks fun! Could you change the letters to "II-FF" instead?
    • II-FF stands for Implementation Intentions for Falls
  • Login looks great
  • Incorrect looks great
    • Could add a "contact my clinician" option
    • Maybe have a place to be able to enter their name and it would be matched with their admin who created/linked to their account
  • Homepage looks great
    • Would the patients be listed on this page for quick access?
    • Maybe a quick view of people who were doing well vs. ones that needed reminders?
  • New User looks great
  • Love the patient page and the charts, note and messages
